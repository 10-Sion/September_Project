회원가입 / 로그인
	탈퇴 신청 중인 status의 로그인 / 회원가입의 제한 : 교직원의 탈퇴 처리에 대한 평가 항목이 있음 
	"관리자는 졸업한 학생 또는 탈퇴 요청한 학생에 대해서는 학생 신상정보를 삭제 할 수 있다." (교수: 해당 내용 동일)
	 - 로그인 회원가입 제한하는 기능 == 불가결
	
	+? ) 회원 가입에 관한 status의 필요성? : 아래 동일 주제로 기술함

교직원
	status : 탈퇴 신청 중 | 재학/재직 중 | 졸업/휴학/휴직 중  => 승인 / 거절/ 상태변경 구현 / 탈퇴시 테이블 상 완전 삭제
	  - 재학과 재직을 하나의 status로 처리하는가? 아니면 학생, 교직원 나눈 세부 상태분류가 더 필요한가?

	+? ) 회원 가입에 대한 status도 필요한가? : 가입 시 인증이나 승인 절차 필요하다 판단됨
		학생, 교수 정보 등록이 수정,삭제,조회 기능과 구분된 것으로 보아 가입 승인 절차 == 요구사항으로 판단됨
		쟁점 : 회원가입과 교직원 경계 애매해져 해당부분 공동작업 or 명확한 분배 필요함
		공동작업 시 회원가입-교직원 | 교수-학생 나눠 코드 병합 작업 하면 될듯

교수
	status : 기존 테이블상 없던 status 추가 : 재직 중/ 휴직 중/ 탈퇴(사직 및 퇴직) 신청 중

	휴직을 단순 신청만 구현하는가? : 세부 구현할 시 기간/ 사유/ 처리 상태 등 필요할듯
	휴강 : 구현할 것인가?

	교수가 새 강의의 개설, 기존 강의 내용 수정의 기능 : 수정의 경우 작성자에게만 권한 줄 필요
	성적 관리도 동일 맥락
	
학생
	status : 기존 테이블의 status 활용 : 재학 중/ 휴학 중/ 탈퇴(자퇴) 신청 중

	휴학을 단순 신청만 구현하는가? : 교수와 해당 내용 동일

	강의 평가 : 해당 과목 수강 / 이력이 있는 학생만(교수, 교직원 제한) 작성 가능하게 : 
		이를 어떻게 구분 ? 강의 테이블의 크기를 수강 최대인원으로 제한 시 : 수강 완료했던 학생에겐 권한이 없음
		=> 본인 생각 : 테이블 크기를 수강 가능 최대인원으로 제한, 강의 평가 작성에 기간 제한을 둠
		== 기간 제한의 구현까지는 필요없어보임 : 해당 정보 안내 정도면 충분할듯

		:: 강의 평가 등록 화면에서 교수에게 이메일 보낼 수 있어야함 (*요구사항)

	수강 신청 결과 처리 : 수강 취소 시 해당 행 날려버리는 경우 = 인원 크기 제한해뒀으면 매번 정렬하는 작업도 필요할듯?
